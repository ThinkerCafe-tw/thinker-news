1. OpenAI重磅推出GPT-5.2-Codex！專攻程式開發、防禦資安，你的AI coding助手升級了！

🔧 分類: AI工具與應用

OpenAI 日前正式推出 GPT-5.2-Codex，這是一款為程式開發者與資安專業人士量身打造的語言模型版本，基於 GPT-5.2 進行特別調校，專注於協助撰寫程式碼與防禦型資安任務。這項升級使開發者在程式碼生成、除錯、自動審查等方面能獲得更高精度的協助，亦強化了模型在資安風險分析、弱點偵測方面的能力。

目前 GPT-5.2-Codex 已在 ChatGPT 的付費 Codex 模式中上線，預計數週內將開放 API 存取，便於企業與開發團隊整合至自身的開發流程中。此外，OpenAI 推出「邀請制可信存取」計畫，優先邀請以防禦資安為主的專業組織進行測試，並提供更寬鬆的模型存取權限。

此模型的推出，進一步鞏固 OpenAI 在 AI 編碼輔助領域的領先地位。對於希望提升開發效率、強化程式碼品質與資安防護的團隊而言，GPT-5.2-Codex 將是一項重要工具。

💡 學習價值：
初學者可藉此工具練習撰寫與優化程式碼，並了解 AI 如何支援資安分析，是進入 AI 開發與資安領域的絕佳切入點。

2. Google再出手！推出超輕量FunctionGemma模型，主打「本機工具呼叫」，手機能變身智慧代理！

🔧 分類: AI工具與應用

Google 近期發表了名為 FunctionGemma 的小型 AI 模型，基於其 Gemma 3（2.7 億參數）模型微調而來，專為「工具呼叫」（Tool Calling）場景設計。FunctionGemma 的關鍵亮點是它能在智慧手機等本地設備上執行，並可依據自然語言指令轉換為對應的 API 行為，如語音控制家電、查詢日曆等。

Google 同步公開了完整的訓練配方，鼓勵開發者依據自家應用的 API 自行微調模型。這項設計兼顧隱私與即時反應，無需依賴雲端，標誌著邊緣 AI 技術的新里程碑。

FunctionGemma 預示了智慧代理的未來方向：裝置端運行、低延遲、強隱私，特別適合嵌入式系統、物聯網與行動裝置應用。

💡 學習價值：
初學者可透過此案例理解 Tool Calling 概念與邊緣 AI 的實務應用，並動手實作語音指令與 API 互動的應用原型。

3. 臺大醫院AI轉型大公開！從「程式做球、AI殺球」到「掛號推薦、照護優化」，GenAI全面進駐臨床日常！

🔧 分類: AI應用實例（醫療）

臺大醫院院長余忠仁透露，目前院內已有超過 300 個 GenAI 專案啟動，其中逾 100 個已投入實際應用。資訊室經理溫家昶形容 LLM 是「生產線的作業員」，強調其與醫療資訊系統（HIS）的深度整合。

這些應用包括：病患症狀分析後推薦掛號科別與醫師、護理問答系統、心臟衰竭照護與醫學影像流程優化等。AI 已成為提升效率、減少遺漏與強化決策品質的關鍵基礎建設，徹底改變醫療流程與服務模式。

臺大醫院的轉型實證了生成式 AI 不僅作為聊天工具，更能深度參與流程自動化與決策輔助，是醫療產業應用 AI 的重要範例。

💡 學習價值：
初學者可從中了解如何將 LLM 與現有資訊系統整合，解決資料分散與流程瓶頸問題，並思考其他垂直產業的應用機會。

4. Anthropic Claude Skills新增「技能目錄」與開放標準，打造跨平台AI代理生態！

🔧 分類: AI平台與代理技術

Anthropic 宣布全面升級 Claude Skills，並推出「Agent Skills 開放標準」，目標是打破不同 AI 平台間的技術隔閡。「技能目錄」讓開發者更易搜尋與部署功能模組，而開放標準則允許同一技能跨平台使用，促進 AI 技能的可重用性與共享。

這項更新讓 Claude 更像一個可擴充的 AI 作業系統，未來若發展成熟，將促成 AI 功能模組的生態系整合，類似 App Store 模式。

對初學者來說，這是理解 AI 如何從「聊天工具」演進為「可執行任務的代理人」的關鍵案例，也有助於認識開放標準在 AI 生態中的角色。
